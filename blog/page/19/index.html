雨中

Sep 5th, 2010 | Comments

昨晚回来的时候已经快12点，下起了很大的雨，天色已晚，冒雨前行，风助雨势，雨越发变得瓢泼，打着伞也不管用了，几乎全身湿透，路上几无行人，可见只有一片茫茫雨幕，只好盯着脚下一方土地步步为营地走。

一个人走在大雨中，一些思绪如暗光中纷飞的雨滴，在脑海中闪烁。

oh，想起来，很久没有更新我的blog了。

1. 雨之印象

近来的多雨天气是由台风“狮子山”带来的，而这个台风的名字，来自于香港一座同名的山。

我不喜欢下雨，它会让我的鞋子变湿。不过雨本身有种干净的感觉。

记得之前同事J在5月的梅雨时节曾经说过，这里的雨多得让人无法理解，记得那时深圳连续下了近一个月雨，几乎没见着太阳。奇怪的是，现在我会联想起J的，竟然是这种无意中提起的小事情。无法理解。这个词语本身就仿佛就无法理解，像一种魔法。

雨的印象还有哪些呢？

一种淡淡气味，来自一个很久没见的女生剪短了的头发。

厄休拉笔下阿纳瑞斯的沙漠。

另一个城市的美丽黄昏，那里有晴朗的天气，夕阳在天边染出一片好看的霞光，虽然很少人留意黄昏的天边美不美。

安静纯粹的睡眠，无意识的黑色世界，永恒之地。

2. 中秋，怀念的季节

两年前的中秋，收到一个HW同事的短信(公司名和人名用了缩写)：

        卜算子　中秋
坂田新天下，锦绣花团簇，中华有为二十载，漫漫辛酸路。
群英鹏城聚，龙凤飞天舞，意气风发尽才智，后继续前仆。

来HW的一些心得，与你共勉，祝中秋愉快。:-) LSH

岁月匆匆，朋友们四散各地，与他也没有保持联系了。只是在两年后这个中秋将近的时间里忽然想起，才意识到，自那之后，我再没有遇到那么有才的人了。

3. 门的启示

据说，当一道门关闭时，另一道门就会打开(When one door shuts, another opens.)，结论就是永远无法关上所有门，对于小偷来说这是个好消息。

我不清楚打开TC这道门对我来说意味着什么，也许只是另一门的关闭。生活处处充满似是而非的暗示，像是一个没有底的迷。

我确切知道的事情有，1. 很忙也很累，2. 并不开心。还有，到了我60岁的时候，如果我有60岁命的话，回过头来看看，我会发现人生中有两件事是确定无疑的。

这个周末，我的门后是双休，Emacs lisp和SICP。

4. 结尾，egg pain未完待续

给我的debian更新了win7的宋体，新的宋体看起来就像无衬线的字体，线条单薄瘦长，像个长身玉立，袍袖飘飘的古人。

回头看看，这篇文字码得很egg pain。

Prefix Notation of Lisp

May 3rd, 2010 | Comments

就像《Writing Emacs Extensions》(Bob Glickstein, O’REILLY 1997)在书中提到的：lisp语言的独特前缀表示降低了常见的中缀表示编程语言在语法上的复杂度。

1. 简化表达式的写法

如对5个数求和，c的写法为

a + b + c + d + e

lisp的写法为

(+ a b c d e)

2. 括号的配对定义了计算的次序，不需要另外定义运算的优先级

相信很多c或c++程序员都记不住全部运算符的优先级。例如，在语句

3 + 4 * 5

中，要定义运算符*优先级大于+。在lisp中，相应的表达式为

(+ 3 (* 4 5))

括号的配对指定了运算的次序。上式与

(* (+ 3 4) 5)

是不一样的。

3. 运算符与函数有统一的语法表示

在c语言中，函数调用的语法与运算符(除了sizeof)不同。例如

4 * 5

执行加法，而

func( 4, 5 )

则调用函数，传递给函数多个参数要用逗号隔开，用于分隔函数参数列表的逗号与逗号表达式

a = ( cal(i), cal(j), i*j )

中的逗号又有不同的意义。

在lisp中，使用运算符与调用函数语法一致，为

(func 4 5)

参数列表中的逗号语法就不需要。类似于逗号运算符的语法表达可使用progn。

简而言之，lisp的语法表达能力相对其他过程式编程语言(如c，或c++，java的对应这部分)并不差–当然c有结构，c++，java都有面向对象，还有佩刀战斗机泛型–其简单一致的语法降低了表示上的复杂度。可使得程序员可以专注于问题域本身，将更多的时间用于分析与设计，而不是语言本身。对比地看，c++其一为人所诟病之处就在于其语言自身复杂度已经很高。

但是统一的或者说是到处使用的括号也使得lisp的代码布满括号，难以阅读。没有括号配对高亮功能的帮助，阅读长长的lisp程序可以说是个恶梦！

给emacs安装info Manual

Apr 18th, 2010 | Comments

发现在debian squeeze(testing)上安装emacs23默认是不带lisp manual的，于是手动安装。方法如下：

  1. 到GNU官网下载info形式的manual,感兴趣的有

    GNU Emacs Manual，包emacs.info.tar.gz (下载地址)
    Emacs Lisp Introduction，包emacs-lisp-intro.info.gz (下载地址)
    Emacs Lisp Reference Manual，包elisp.info.tar.gz (下载地址)

  2. 安装下载的info包。debian下标准的info文件位于目录/usr/share/info，此目录下文件dir用于提供info树的根菜单，此目录下每个info包在文件dir里都有一个条目对应，条目可以放在一个或者多个section下，每个条目由三部分组成

[条目名]: [对应的info文件或包(可选的子条目名)].  [条目描述]

条目名后面要带一个”:”号，文件或包名后面要带一个”.”号。如emacs faq的条目为

* Emacs FAQ: (emacs-23/efaq).   Frequently Asked Questions about Emacs.

将下载的三个包放入/usr/share/info/。并在info树根目录增加中相应的节点，即在文件/usr/share/info/dir的Emacs section中加入

* Emacs
...       <== 其他条目

* Emacs Manual: (emacs.info.tar). GNU Emacs Manual
* Emacs Lisp Intro: (emacs-lisp-intro.info).  An Introduction to Programming in Emacs Lisp
* Emacs Lisp Reference Manual: (elisp.info.tar).
                                GNU Emacs Lisp Reference Manual
...       <== 其他条目

即可。也可以安装到自定义的info目录，此目录下要有dir文件，同时添加此info目录到INFOPATH来让info进行查找。对于bash，添加环境设置到.bashrc：

INFOPATH=$INFOPATH:$ADDED-INFOPATH
export INFOPATH

对于emacs，可通过变量Info-directory-list, Info-default-directory-list来定制，具体参照Manual。

如担心手动添加info条目会损坏原文件，可先做备份再修改，或通过install-info命令添加info条目，如

$ install-info --dir-file=dir --info-file=filename.info --section=section-name --name=entry-name --description=entry-desc
