MySQL Test Suite

Sep 1st, 2011 | Comments

1. 介绍

在寻找MySQL测试程序的过程中，发现它的源代码中就自带测试程序，一套叫做MySQL Test Suite的包括二进制文件，脚本和测试用例，输出结果的测试框架。
关于这个MySQL Test Suite，《深入理解MySQL》有详细的描述，参考
http://book.51cto.com/art/201001/180745.htm
和
MySQL自动测试框架介绍，http://rdc.taobao.com/blog/cs/?p=480

2. 使用

2.1 系统apt源安装包中的Test Suite

通过apt安装的mysql-server，这个mysql-test-run.pl会被安装到系统目录的
/usr/share/mysql/mysql-test/
运行此脚本会报错

$ ./mysql-test-run.pl
Logging: ./mysql-test-run.pl
mysql-test-run: *** ERROR: Could not find any of /usr/share/client_release /usr/share/client_debug /usr/share/client/release/ /usr/share/client/relwithdebinfo/ /usr/share/client/debug/ /usr/share/client /usr/share/bin

此脚本会假设../client目录下有mysql客户端程序，而通过apt安装的mysqlclient的目录layout与它的源代码目录的layout不一致，因此不能正常工作。

2.2 源代码库中的Test Suite

2.2.1 下载源代码

于是直接下载MySQL的源代码。在MySQL的官方网站上
http://dev.mysql.com/downloads/
点击社区版的”MySQL Community Server”下载链接，这时最新的版本是5.5.15，在平台选择下拉列表中选择”Source Code”，再选择Generic Linux TAR Archive包开始下载。

2.2.2 编译源代码

安装步骤和过程参考MySQL 5.5的官网文档
MySQL 5.5 Reference Manual, 2.9.2. Installing MySQL from a Standard Source Distribution
http://dev.mysql.com/doc/refman/5.5/en/installing-source-distribution.html

在linux平台下编译要注意，与5.5之前的版本用GNU AUTOCONF和AUTOMAKE编译不同，版本5.5将各个平台下的编译工具统一为CMake(CMake，Cross-platform Make是一个强大的跨平台开源编译工具)。所以编译的步骤有一点不同。

对比MySQL 5.1 Reference Manual，5.1的编译步骤是：
(参考http://dev.mysql.com/doc/refman/5.1/en/installing-source-distribution.html)

# Beginning of source-build specific instructions
shell> tar zxvf mysql-VERSION.tar.gz
shell> cd mysql-VERSION
shell> ./configure --prefix=/usr/local/mysql
shell> make
shell> make install

而版本5.5编译步骤是：

# Beginning of source-build specific instructions
shell> tar zxvf mysql-VERSION.tar.gz
shell> cd mysql-VERSION
shell> cmake .
shell> make
shell> make install

当然要顺利编译成功，需要安装cmake，在ubuntu下通过apt源安装cmake很方便:

bash# aptitude install cmake

在ubuntu下编译cmake那一步遇到问题，报错如下：

-- Check size of wint_t
-- Check size of wint_t - done
-- Could NOT find Curses (missing: CURSES_LIBRARY CURSES_INCLUDE_PATH)
CMake Error at cmake/readline.cmake:83 (MESSAGE):
Curses library not found. Please install appropriate package,

remove CMakeCache.txt and rerun cmake.On Debian/Ubuntu, package name is libncurses5-dev, on Redhat and derivates it is ncurses-devel.
Call Stack (most recent call first):
cmake/readline.cmake:127 (FIND_CURSES)
cmake/readline.cmake:217 (MYSQL_USE_BUNDLED_LIBEDIT)
CMakeLists.txt:257 (MYSQL_CHECK_READLINE)

-- Configuring incomplete, errors occurred!

发现它依赖于ncurses库，于是通过apt源命令行安装:

bash# aptitude install ncurses-dev

ncurses-dev是一个虚包，实际安装的包是libncurses5-dev。然后清理之前出错的临时输出:

bash$ rm -f CMakeCache.txt

这一步相当于”make clean”。然后重新编译

bash$ cmake .
bash$ make
...

2.3 简单使用介绍

编译完源代码之后，可以直接运行mysql-test-run.pl，切换到源代码根目录下的mysql-test目录:

bash$ cd mysql-VERSION/mysql-test

通过命令行

bash$ ./mysql-test-run.pl --force

自动启动mysqld并运行所有测试用例，–force选项相当于make的–keep-going选项，不加此选项的默认情况，mysql-test-run.pl脚本在首次遇到测试错误时就会中止，不运行后面的测试用命，添加此选项后即遇到单个错误也不会立即中止，继续运行后面的测试。
如果需要有选择性地只执行部分测试用命，可通过–suite选项指明需要执行的测试类别，如输入命令

bash$ ./mysql-test-run.pl --suite=main

只执行目录t下的所有测试用例，还可以通过指明单个测试名来执行单个或多个用例，如输入命令

bash$ ./mysql-test-run.pl alias analyze

只执行alias.test和analyze.test两个测试用例。 对已经启动的mysql服务器(mysqld)进行测试，可使用命令：

./mysql-test-run.pl --suite=main --force --extern sock=/var/run/mysqld/mysqld.sock --extern user=test_user --extern password=test_user

其中–extern用来将特殊选项的值传入，sock指向mysqld启动打开的sock，user和password分别指定连接数据库服务器的用户名和密码。

“I have always said if there ever came a day when I could no longer meet my duties and expectations as Apple’s CEO, I would be the first to let you know. Unfortunately, that day has come.”

别了，乔帮主。

当我谈机械键盘时，我谈些什么

引子：天涯夕阳红，剑客论道时

大漠。
烈风如刀。

已到黄昏时分，火一般的红日缓缓落向西边，染红了西天的云彩，仿佛半个天空的云彩都被点燃了一般，流霞如幻，飞云炫目，好灿烂的一副光景。 风突然缓下来了。
飞舞中的黄沙也随之慢慢安静下来了，扬在空中的沙尘开始飘落，散于地上，那流动的黄色映照着红艳的夕阳，如丹沙，亦如鲜血！

黄沙落尽，沙地现出两个人影。
两人在距离不远的地方相对而立，其中一人白衣如雪，脸色如玉，两手垂立，一把长剑另在腰间，另一人青衣方帽，剑眉星目，两手束于身前，宽大的衣衫飘飘荡荡，却看不出有否带兵器。但这两个人都有一个共同点，那就是眼睛都紧紧盯着对方！

青衣人先开口了，“听说最近江湖上出现了一个人，每次出现都从东方而来，不知道你可认识？”
白衣人不露声色地说：“听说江湖上有一个人，发掘了道之源头，江湖上各门各派都由此而生，不知道你可认识？”
青衣人眼神闪动，似乎在仔细打量眼前的人，说：“想必你就是那个从东方来的人。”
白衣人说，“想必你就是那个发掘了道之源头的人。”
一阵微风吹过，带起两人的衣襟，青衣人双手藏在随风滚动宽大衣袖里，白衣人的白衣显得更加飘逸了。

青衣人问：“你为何而来？”
白衣人答：”为道而来。”
青衣人再问：”道在何处？”
白衣人答：“道在剑上。”
“剑可以用于防卫，也可以用于杀人。”
“据我所知，用于防卫的剑少，还是用于杀人的剑比较多。”
青衣人微笑，说：“听说你有一把好剑？”
白衣人抽出腰间的长剑，平托着向前半伸手臂，似乎是向青衣人展示自己的武器，然后说：”一流工艺，新颖设计，历经3代近10年的演进，才有了我手上这一把‘忍者’”。
在落日光线的照射下，剑上filco的标志闪烁着似灰似黑的金属光芒。
青衣人看到这样一把好剑，满面欣赏之色，不禁称赞道：“果然是一把好剑”。
白衣人将剑收回左手，问青衣人：“听说你也有一把非常厉害的剑？”
青衣人说：“的确，曾经我有一把很好的剑。”
“好在何处？在下愿闻其详。”
“诞生最早，专注品质，坚持30多年的用心，才能打造出那样的一把’g80’。”
白衣人眼睛一亮，说：“如此一把好武器，可否拿出来，让在下见识见识。”
“但那把剑现在已不在我手上了，我已不再用那样的武器。”
“对于一个剑客来说，能得到一把那样的剑是梦寐以求的事，我想像不到有什么原因会使你放弃它。”
青衣人展颜一笑，“因为我发现了比那把剑更厉害的，比所有的剑都更厉害的，世间最厉害的武器。”
白衣人愕然，喃喃道：“比所有的剑都更厉害的……世间最厉害的武器……”，思量沉吟半晌，然后眼神锁住青衣人，问：“那是什么？”
这时青衣人笑而不语，依然神秘地把双手收藏在宽大的衣袖中。突然，又一阵微风吹过，吹起了地上的细尘，吹动了两人的头发和衣衫，吹起了青衣人宽大的袖袍一角，而就在这电光火石的一刹那，青衣人迅速张开了那随风飘动的袖袍，向白衣人的方向扬去！
白衣人呆住了，没有动作没有反应，就在这一刹那，他已经看到了青衣人所说的，比所有的剑都更厉害的武器，世间最厉害的武器，就在青衣人那衣袖的一扬之中，他已经看到了，也已经领悟到了。

再谈机械键盘

上面仿古龙武侠体写了段两个机械键盘大品牌filco和cherry比较的小故事，应用中小学时候学到的40分钟作文大法中的一招：3段式作文，先写一个开头，在悬念中东扯西扯进入正文，谈谈机械键盘。

入手cherry g80黑青后，我发现用它来写代码手感真是越来越欢畅，虽然说几个卫星轴大键还是处于新键盘磨合期没用开，但青轴爽朗的打击感和强烈的段落感还是让手指迷恋不已。而在用了一段时间之后，发现原来的choc mini这款键盘的问题还是比较多的，举例其中比较蛋疼的几点：R4区键位排列比较密而且边框高，左上角的esc如同深陷边框中，这一排键盲打经常按错键，好在我是emacs党而不是vi党，不然不改esc键映射没法活，而且键帽印字很淡，看起来也很模糊。做工比较差。键帽既不完全是cherry的原厂高度，也不是OEM的高度，这样就不能换键帽，基本上失支了机械键盘的键帽可玩性，坏了键帽也只能找售后。后来把手上的choc mini黑茶出给了本地的一个二手键盘玩家，然后把cherry g80拿到了公司办公室替换原来的choc mini，周围的同事看到，纷纷来问，当听到价格之后，又纷纷表示没看出这么土的键盘有哪里贵的竟然值几百块。结果就是，从此以后这个并不安静的办公区里，每天多了一些敲打青轴键盘的“咔嗒”声。

上周在天下外设论坛上闲逛时看到帖子说，filco二代87白色青轴的国内行货在18号正式到货开卖，于是第一时间在国内的总代花尽旗下的键步如飞淘宝店下了单，在上周六20号下午，从顺丰快递拿到了这首批二代87青国内行货中的一把。开箱后整体看了一下，奶白带黄的材质比较细腻，有点像是瓷而不那么像塑料，键帽依旧小巧而涂层不那么明显，听说二代的涂层较一代有改进，手感更好而且不那么容易打油，不知道实际用起来情况如何。至于铭牌的话，87的铭牌放在方向键的下方，二代的铭牌是灰色的。其它的像键盘线，防尘罩那些与一代没有太多变化。

键皇到手，第一时间把键帽换了，换上从二嘟那里搞回来的全副淡彩pbt键帽。换键帽是个体力活，整个键盘的键帽全换更是一个体力活。花尽随键盘送了一个filco二代钢丝拔键器，用它来拔键虽然速度比较慢，但是它的力量传导比较好，拔起键来并不费劲，也不会伤到键帽。同时由于原来的键帽是abs，拔起来也很轻松顺利，对比拔cherry g80新键帽的时候大力一拔把卫星轴钢丝连带飞出的恐怖情景，换这个filco的键帽可谓是手到擒来，波澜不惊。最后换键帽完成，眼前得到一把filco二代87白青+87键淡彩pbt键帽的顶级奢华搭配，可以直追realforce。如此华丽的新搭档应该上图留念的，不过手上没有太好的拍照工具，暂时按下不表，日后再上图了。

filco这个品牌，其实它的配置性价比并不高，最为人诟病的一点就是abs键帽，无论是原厂的pom或者国产kbc/ducky的pbt或plu的pom都比它要好，不过综合来说，它的做工比国产品牌好，这两个的比较有点像apple notebook与神舟笔记本电脑做对比。而另一方面，它继承了日本产品精巧细致的特点，窄边框小面积的造型设计与原厂的传统大气走的是完全不同的路线。在国内用filco键盘的人关注它的人，还是挺多的，不过价格较贵是事实，当然市场价格受各种因素影响，这里面的水太深，简单的说，反正日本的电子产品来到中国卖就是贵，但中国人均收入远没有日本人高。

对比两个青轴键盘，原厂的键盘没有钢板，虽然说是青轴段落感很强，但触底的时候会感觉偏软，回弹也没有带钢板的有力，但是这种硬中带软的不会像薄膜键会觉得回弹无力手感枯涩，相反由于键帽底下有mx轴，回弹还是能感觉到弹簧的力道的，而filco的键按下回弹比原厂有力迅速，声音也相对响亮清脆一些，它的触底比较硬，所以使用起来感觉filco的键程比原厂的短，但实际上两种键盘用的都是同一种轴。

应用40分钟作文大法，后面一节续上开头古龙武侠体小故事的结尾，最后篇末点题，说说我对机械键盘，或者对所有工具的一种看法：工具是人四肢五官的延伸，工具的灵魂来自于人。所以工具是什么样的并不重要，重要的是人的本身，用这种工具去做什么样的事情，体现了什么样的价值和意义。

终章：道现人散去，天地两茫茫

夕阳西下，天色渐暗，沙漠的夜晚很快就会降临。
风势渐变疾劲，风夹黄沙，吹浓暮色，吹得天地一片茫茫。
青衣人渐渐走远了，他施展轻功，边吟边歌，很快就远去变成一个模糊的身影，一个小点，最后完全消失在夜色中了。
在这边四方无人烟的沙漠之上，只留下白衣人，还一直木然立在原地。
他有所领悟，心有所得，面对着白衣人离开的方向，又似乎略怀惆怅，茫然若失。
因为青衣人拿出了比所有的剑更厉害的，世间最厉害的武器，因为他领悟到了，也知道了青衣人的意思，因为青衣人向他展示的，也就是他来此地目的——最高层次的道。
因为青衣人向他展示的，竟然是青衣人的双手！
因为一个人的双手就是比所有的剑更厉害的武器，一个人的双手就是世间最厉害的武器！

人生的无奈与美好

今天约了海潮他们踢球，遇上一队身体素质和配合都不错的对手，被虐全场。踢完球离开华工，在校门前面的食街找地方一起吃饭，老地方韩国菜，记得这回是海潮毕业工作之后第一次请大家吃饭，也算是庆祝上月第一次出粮。席间说天论地，风生谈笑，吃完饭出来，一行人走在路上，当其时正在校门大石头附近，一旁的食街商铺灯光闪烁，不远处的校园沉默在夜色中，周杰说他很快就会离开广州去珠海，以后和我们各分两城，要抓住不多的机会一起再踢踢球。突然间似乎觉得有点感触，脑海中想起很多片断：大三足球比赛夺冠一起吃夜宵半夜走回宿舍，一群人喧哗而过的大学城大马路；毕业时各奔东西的匆忙；夏天午后深大的安静，热天夜晚深圳科技园整齐的道路和大冲拥挤的人流；最近一段时间相约在校园西区球场踢球的雨天和晴天。才发现，原来时光已经带我们不知不觉地走过这么多年，走了这么远。

周杰的想法，我们都知道已久了，与很多在毕业后依然在一起但分居两地的情侣一样，他与女朋友也一直面对着地理阻隔这个难题，到今天他终于做好准备动身去珠海与女友一起生活，也算圆了一个心愿。

虽然说最近一段时间也不时的约他们踢球，也一起吃饭，也经过这些地方，但平时大大咧咧后知后觉地活着，也没有意识到，原来这熟悉而平凡的一切，也有一天会失去会变残缺，只是在有人将离开的时候，才特别的觉得要珍惜。

回家的路上，夜色慢慢变深，缓缓前进的公交车外，广州城中的路灯相互交替，映射出灯光从远处渐渐拉近，接近而又渐渐远去，望着窗外的夜，思绪如野马乱奔。

在我这个局外人看来，从分居两地走到一起生活，对于男女双方来说无疑是一种幸福，但是否也是人生的一种无奈？在这个时代这个节骨眼上说，我们80后这个群体最深刻地感受到时代改变的影响，从小到大很多不好的事情都上我们赶上了，到如今高房价，生活压力大，大学毕业生出路少等等，这些常常在电视报纸，新闻报道中出现的字眼和问题，终于有一天会间接或直接地影响到我们每一个人的生活。广州房价高，也许不是每一个人都有钱买得起，也许即使有了钱，也并不是每个人都希望继续留在这个制度依然陈旧分配愈变不均压力愈变大的城市。作为一个个单个的社会个体，细小的个人面对庞大的社会机器，终究无力改变太多，只能无奈的接受。面对这个时代的无奈，今天周杰做出了他自己的一个中长期选择，而更多的80后朋友，他们如何做出他们各自的选择？

从个人的角度来看，一个男人为了另一个女人，放弃他已有的事业，到另外一个地方去重头开始一切，对于这个男人来说，获得的幸福算不算也带着一份牺牲？如果这个故事里的男主角换成了我，我会因为一个女人而付出这么多吗？伟大的爱情是不是本来没有包含太多的计较，只有一味的付出和给予？周杰的确是找到他的幸福了，而到今天我仍然没有遇见属于我的那个女主角，没有可以做如此付出的对象，其实是不是可以说是一种失败或者悲衰？人生因无奈而不完美，但又同时因为爱情而变美好，在这个世界活了四分之一个世纪，我没有去跑步，却发现人生是这样混着无奈和美好的一种滋味……

很多问题也许没有所谓的答案，正如人生，如果它算是一个问题的话。人一辈子来世上走这么一遭，也许答案与否什么的并不重要，埋入黄土的是不是答案也不重要，也许并不是每个人都能找到所谓答案，但至少每个人都经历过寻找的过程。

新博客安家

Aug 21st, 2011 | Comments

虽然我的博客平时更新比较慢，访问量也小，但是有一个比较好用的博客还是一件赏心悦事，之前在is-programmer.com上写博客也有一段时间了(记不清是1年多还是2年)，相对其它较大的程序员bsp来说，它比较小众，有不少功能亮点，虽然它的作者galeki也一直在维护更新，添加功能，在我使用一段时间后，总的来说感觉还是限制比较多不够强大，为了更方便的功能和更高的可玩性，终于还是自己搭了一套新博客。用的是流行的wordpress系统。

用wordpress搭建独立博客，要准备独立的域名和可用的主机空间。我在GoDaddy上申请了一个域名onhacking.com，主机空间用的则是戈戈主机的，性价比相对外国的一级主机供应商来说不算好(空间小，而且有内容审查和流量限制)，但是只运行我这么一个小博客足够用了，成本比较低。在这上面搭一个空的wordpress框架，简单配置一下，装几个常用插件，博客就可以跑动了。

然后想到把旧的博客上面的文章迁移到新博客上，遇到问题了：is-programmer.com上的博客没有导出功能，wordpress上默认的导入功能只有Blogger，RSS几个选项。怎么把旧博客上的文章导入到新博客上呢？旧博客上的文章虽然不算太多，但粗粗看了一下，也有几十篇，如此多的数据如果是手动一篇一篇的copy-paste是行不通的。后来在网上找了一下，发现这个博客迁移问题也是很多人会遇到的常见问题，目前国内有这么多的bsp，因为某种原因换bsp也是很常见的事，但是一旦换bsp，就面临着博客迁移的问题，有一些博客搬家工具如blogbus专门针对这个问题提出了解决方案，但是这工具也有不少的问题，比如说只针对部分bsp有效，迁移时做不到全部内容的迁移，收费等。收费也许对那些有钱人来说，也许算不上一个问题，但是对一些人还是会形成阻碍，因为大部分的bsp服务都是免费的，迁移免费的博客要收费，这并不是很好的解决方案，所以有的bsp也推出免费的博客迁移功能。对于我的问题，这些工具的主要问题是不支持，is-programmer.com是一个程序员的小众博客，用户市场不大，还没有哪个博客搬家工具对它有支持。

套用搬家工具的实现思路，我后来想到的解决办法就是把旧博客导出为RSS，然后导入到wordpress中，现有的软件(工具)不支持，那就根据自己的需求写一个新的，典型的实用型程序员思维。由于网上找到的这些搬家工具都是闭源的，没有太多可以直接拿来参考的代码，于是一切从头开始——亲，真怀念开源世界的好。于是我这个脚本党开始动手了，主要步骤有3个:

  1. 从旧博客导出。is-programmer.com没有导出功能，只能另外做一个。一个博客迁移，需要导出的数据有哪些呢，文章，评论，链接，留言，版面相关的一些设置等。对我来说，主要是文章，其它方面的数据量很少，可以直接无视。写了一个脚本把旧博客上的所有html页面全部抓了下来，python里有强大的类库urllib，urllib2，实现这点功能相当容易。
  2. 数据格式的转换。把抓下来的html内容转为RSS 2.0格式，除了文章之外，其它内容如评论和留言数量很小，就不另外折腾迁移了。再一次python脚本，读入解析源html，最初用的是HTTPParser，可惜它的功能太弱，转为re，虽然逻辑写起来麻烦了一点，但是胜在表达力强。生成RSS 2.0格式文件，最初用ElementTree，功能也太弱，后来用lxml.etree，接口也同样简单好用，功能较为丰富。
  3. 导入将生成的RSS文件导入到wordpress中，这个直接用wordpress的功能就可以了，不过导入过程遇到文章的时间格式不对，wordpress的正文标签(content:encoded)和RSS标准的(description)不一致等一些小问题，小解决一下，文章导入完成。
  4. 从旧博客拉取文章引用的图片，文件等内容，上传到新博客空间。再一次脚本，有wget，curl这些基础工具，做起来相当easy。

于是一轮折腾之后，就导入了旧博客的所有文章，再调整一下版面主题，虽然好多地方还是比较粗糙的，但大体上OK了，新博客的搭建告一段落。 接下来的就是，有空写写博文，有空玩玩wordpress，有空的时候再想想有空的时候可以干嘛，上线，运行，再运行。

这里感谢每一位读者过去对我的博客的支持，虽然你们和我多未谋面，有联系过或者没有联系过，希望这个本着记录观点，分享知识的想法而写的新博客也能得到你们的喜欢，感谢支持！
