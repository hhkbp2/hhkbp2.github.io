A Thought on Bit Representation of Primitive Integer Type in C/C++

Nov 28th, 2010 | Comments

老的space live博客要迁移wordpress，不迁移的都要杯具了，保留两篇以前的文章转过来。

看天下大势如此，营利的博客业务开始衰落了。

2009/8/7 13:34:10

<<C++ Primer>> mentions when talking about primitive integer type as follows:

For the unsigned types, a negative value is always out of range. An object of unsigned type may never hold a negative value. Some languages make it illegal to assign a negative value to an unsigned type, but C++ does not.

In C++ it is perfectly legal to assign a negative number to an object with unsigned type. The result is the negative value modulo the size of the type. So, if we assign 1 to an 8-bit unsigned char, the resulting value will be 255, which is 1 modulo 256.

at page 37 (<<C++ Primer, 4th>> English, Posts & Telecom Press, 2006)

Why a negative number which is assigned to a unsigned type would become a modulo of it? Is it because of the bit represatation? To validate the situation, I write this short code fragement(take char for example to figure out the bit representation):

//: bit_inter.cc
#include <iostream>
using std::cout;
using std::endl;
#include <climits>
template <typename T>
void print_binary(const T tchar)
{
for(int ix = sizeof(tchar)*8 - 1; ix >= 0 ; --ix)
{
if( tchar & (1<<ix) )
cout << "1";
else
cout << "0";
}
}
#define PRINT_BIT_INTER(x) \
cout << (int)x << "\t" << " binary representation: " << "\t"; print_binary(x); cout << endl;
int main()
{
unsigned char uchar;
signed char schar;
cout << "the limit of type char and unsigned char:" << endl;
cout << "size of signed char: " << sizeof (signed char) << "\t" << "size of unsigned char: " << sizeof (unsigned char) << endl;
cout << "SCHAR_MIN: " << SCHAR_MIN << "\t" << "SCHAR_MAX: " << SCHAR_MAX << "\t" << "UCHAR_MAX: " << UCHAR_MAX << endl;
cout << "the representation of type char and unsigned char:" << endl;
schar = 127;
PRINT_BIT_INTER(schar);
schar = 0;
PRINT_BIT_INTER(schar);
schar = -1;
PRINT_BIT_INTER(schar);
schar = -127;
PRINT_BIT_INTER(schar);
schar = -128;
PRINT_BIT_INTER(schar);
uchar = 0;
PRINT_BIT_INTER(uchar);
uchar = 128;
PRINT_BIT_INTER(uchar);
uchar = 255;
PRINT_BIT_INTER(uchar);
return 0;
}

//* output:(match 100%)
the limit of type char and unsigned char:
size of signed char: 1  size of unsigned char: 1
SCHAR_MIN: -128 SCHAR_MAX: 127  UCHAR_MAX: 255
the representation of type char and unsigned char:
127      binary representation:         01111111
0        binary representation:         00000000
-1       binary representation:         11111111
-127     binary representation:         10000001
-128     binary representation:         10000000
0        binary representation:         00000000
128      binary representation:         10000000
255      binary representation:         11111111

*///:~ bit_inter.cc

now the bit representation of type char set is clear\:

bit representation    unsigned char    signed char:
11111111              255                  -1
10000001              129                  -127
10000000              128                  -128
01111111              127                  127
00000001              1                     1
00000000              0                     0

so we could conclude that the same bit representation gives different value when representating in different type, signed char or unsigned char.

when assigning a negative number, e.g. -1, to a unsigned char, e.g. named uchar, what C/C++ actually does is keep the bit represetation of -1 intact and copy it into the location where the unsigned char uchar lies. So by the way of unsigned char value representing, the unsigned char now have the value 255. It seems C++ have calculated the modulo, -1 % 256 = 255 for us.

Like a Rolling Stone

Nov 24th, 2010 | Comments

晚上回到住处，打开google音乐播放器播歌，然后在床上躺了一会儿，不知不觉睡着了。醒来的时候电脑在播bob dylan那首著名的歌，”How does it feel? ah, how does it feel? To be on your own, … , like a rolling stone …”，突然想起以前的老板martin，记得他说过，他喜欢听的是bob年代的音乐，耳边的歌声好像也变成的martin的声音，顿时百感横生。

japhen离开深圳一起吃饭的时候，说起我离开后，以前的一帮人走的走散的散，短短一段时间变化很大，虽然很多东西我不能左右，不过如此听来隐隐还是觉得，我伤害了很多人。

也许这时我需要啤酒。martin说过，生活太杯具(horrible)，人生不如意的事很多，喝酒能让人忘记不开心的事，很多英国人都习惯下班去喝酒，喝到醉了就回家睡觉。在我这个80后中国人听来，如此的人生仿佛很是无奈，不过认真想想，人生短短几十年，又有谁弄得清楚有些什么意义？终日为声色名利无止境地奔波劳碌，如能在某个夜晚，或约上三五知己，在昏暗的酒吧里喝上几杯，在这样的时间里，所有的历史与未来，浮华与潜流，伟大理想与卑鄙阴谋，激烈动荡与和平演变，冰与火都会悄然远去，一个完全属于自己的夜晚，又何尝不是一种特别的体验？

不知道老朋友们最近有没有去喝上一杯。

在TC的离职流程已经开始走了，今天一天只走了HR离职面谈一步，看来很多事情都有向后拖的趋势，我还要去多混几天。今天也收到f的电话，叫我抓紧回广州面试了，他一再解释说，新岗位前景有吸引力，好像怕我不放在心上似的，其实我完全没有看不起他们公司的意思，因为我也不是那种很牛的人或目空一切的人，主要是我想在离开TC之后，再计划下一步。其实一边先面试一下也无不可。有时会觉得，历史会重演，人其实是没有在教训中变得聪明，世事变幻，物事人非，其实很多事情想来也没有所谓答案不答案的。

在付出我的代价交出我的学费之后，我学到一些东西，有些在工作与技术方面，更多的在生活方面，了解到在深圳这个地方有一些人原来是这样的工作并生活着，唐皇说以史为镜，以人为镜，意思可谓相当深奥了，而当镜子中的我头上的头发慢慢变少，人生的经历却慢慢多了，无论这些经历是好的还是坏的。很多东西旁人只会从社会的角度来看，不能说这种角度是错的，更多的时候，我只是觉得无需理会其他人的指指点点，按自己的路走下去便是了，人生只有一个，每一分钟是完全属于自己的，无论你有多少情人好友，这一路的悲欢苦乐，更多的时候都只能孤独地领受。这种意思铺展开来，就像是抄袭了古龙小说中对人物的描写叙白了。

在反思过去的时候，发现其实我的时间其实可以用得有意义很多，想想如果这几年来耗在qq上的时间，如果有一半用在做正事上，可以看很多书，写很多代码，做很多有益身心有益社会的事，那世界又会是什么样子，转到linux后已是日渐少用它了，于是决定不再用了。在qq的个性签名上留了一个邮件地址，就算是留一个联系方式，做了一个简单的通知和了结，签名后面加了这样一句，”Somewhere along the line I knew there’d be girls, visions, everything”，这句话来自《on the road》第一章结尾的一段。

弃用qq与前段时间的互联网大战无关，这件事自始至终我发过意见什么的，即使在战况最激烈公司群上各处论坛人声鼎沸的时候也没有趁热闹发过言。有亲友问到，简单交流几句，如此而已。套用黑泽明电影《乱》的一句对白，”这是个弱肉强食的时代”，而在21世纪这个时代这个神奇的国度真是有很多神奇的事情会发生，linux用户保持蛋定，对我来说，多用电话和邮件，加上各处大大小小的USENET，irc，论坛一堆，足矣。farewell, weapon. farewell, qq. 时间永恒流逝，时代不断变化，但总有一些东西是保持不变的，比如bob dylan的歌，比如jack kerouac的小说。

其实《on the road》第一章结尾那段还有一句，”somewhere along the line the pearl would be handed to me”。

是的，我还年轻，我渴望在路上。
