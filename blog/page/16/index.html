杯具的诞生

Nov 9th, 2011

今天这篇日志主要是讲一个故事：

从前有个阿叔，他叫本华，有一天他将系统升级到了gnome3，具杯的使用体验让他痛苦不已，促使他开始思考杯具的本质，然后他着手写了处女作品《具杯的诞生》，成为一代哲学家。

(故事完，热烈掌声响起，一浪接一浪，一浪接一浪……)

我的debian系统是testing/sid混合源，由于平时更新勤快，所以一般每次更新的更新包都不多，昨晚因为gnome-panel上声音控制的applet启动加载有错，添加applet的对话框也找不到那个applet，尝试卸载并清空配置重装gnome-applet的时候，一不留神使用了apt的-y选项，然后去喝杯水打个转回来，发现已经更新了几百M的包，重启之后就已经是gnome-shell接管了桌面，然后更新管理器提示更新3.0内核等等，再来一次更新个几百M。

第一次与gnome-shell亲密接触，才发现它的用户体验是既坑爹又坑哥，与gnome2的习惯相差太远，抛开黑乎乎空荡荡的上边栏不说，抛开横竖让人找不着北的菜单栏不说，抛开迟钝的界面响应和突然死亡的X不说，也抛开各种华而不实的效果，但无论如何，关机键要给哥留一个吧，难道真的要我看着”Suspend”菜单按Alt键，或者直接命令行shutdown？ 这个设计也太反用户，反人类了吧，让思念关机键的哥情何以堪？

在此具杯之下，我只有一句话想说:“Well done，gnome3，你已经成功让我恶心到了”。在你统治的X下我会业业兢兢，学会即时保存所有的文本，在2012到来之前学会欣赏你的突然死亡特效，学会习惯于忘记马上卸载掉你并找回你前任gnome2的想法并假装与你相处得愉快幸福，最后要学会的是带着期待生活，期待有一天，经过改进的你真的让人在使用中体验到方便愉快，然后让人不得不称赞说：

“Well done, gnome3.”

阳朔之行

Nov 3rd, 2011

9月中旬的一个周末，去了一趟阳朔游玩，行程是由公司组织的，前后三天，时间比较短，很多风物只能算是匆匆一瞥，只留下了一个印象，旅游回来之后，想写写路上见到的一些风景和感受，但一直没有把握好时间。现在回头看看也过去一个多月了，算是补交一下功课，这里小写一下阳朔一行。

认识中的桂林

像我们80后的这一辈人，认识桂林是从小学的课本开始的，记得小学那个时候有一篇课文叫做《桂林山水甲天下》，讲的是桂林山水的美丽，那篇课文具体的内容是完全不记得了，不过把桂林山水的美描述到超凡脱俗的那种感觉，经过这么多年记忆依然清晰。就像那个时候的一首歌唱的那样，想去桂林可是有时间的时候没有钱，有钱的时候没时间，对我来说总之一直是既没钱也没时间，桂林我始终没有去过，空留一份由书本描述得来的想象。认识中的桂林是美丽的，而且没有去过的地方，总是特别的美丽。

漫走阳朔

从广州到桂林坐汽车路程有600多公里，我们是下午出发，坐了7,8个小时的汽车后才到达阳朔，那时已经是晚上的10点多11点，当晚即入住西街的旅馆休息，第一天自由活动，旅行社的导游少不了向我们推荐套餐a套餐b，但由于觉得这些套餐往往不是太贵就是时间安排太紧密，所以最后还是决定自己走，一群人租了自行车在阳朔县城附近漫游。

骑着自行车从阳朔县城出来，一路慢骑，边看风景，看到阳朔县城不大，感觉上更像一个大一点的小镇，房子普遍都建得比较有当地的特色，平顶飞檐，一般也就4到5层的样子，镇上到处可见拔地而起的陡峭山峰，也比较有特色，而房子靠山而建，两者之间的距离也特别的近，仿佛住在楼上的人伸手即可以触摸到陡峭的山岩。也许是旅游业发达的缘故，酒店饭馆特产店之类的就特别多，游客来自世界各地，走在街上经常看到黄发碧眼的外国人。在县城的街上走，也能感受到此地山水秀丽，因为抬头远望就能看到绵延的青山，县城边上就是清澈的小河。

在阳朔县城附近有很多的景点，这些景点都围绕着当地特色的喀斯特地貌来开发，我们也逛了几个，像聚龙潭，大榕树等等，看看溶洞奇石，看看古树远山，匆匆览过如走马观花，当地的风景的确养眼。在阳朔县城走着，会觉得当地的旅游业相当发达，人流一旦多起来，就会有一种繁华的印象，但到了真正骑着车走到县城外面，才看到原来很多地方其实也是绿树田野，小村小镇，感觉和老家小镇差不多，并不算太发达。景点附近游人众多，是比较热闹的，骑车走远一点，沿着农村的小路慢慢走着，小路绵延，村野安静，山水清朗犹如画卷一样在面前展开，又会另有一种纯朴自然远离繁嚣的感受。

晚上回来，洗过澡吃过饭，和两个朋友逛了一下西街，这是当地的一条比较出名的步行购物小街，西街之于阳朔可比上下九之于广州，小街不大也不算长，开了很多小店酒吧，一到晚上人就特别多，人流密集热闹非凡，也算是当地一景吧。

漓江印象

第二天是公司组织的活动“徒步漓江”，虽然说的好像是走完漓江全程，不过其实只是选择了阳朔附近供游客通航的一段来走，公司组织的活动中间过程相当坑爹情节复杂此处跳过1300字略去不提，但也算近距离游览接触了一下漓江。

感觉中的漓江不会是那种很深的江，而从游览过的这一段来说，江面不大而比较开阔，江水舒缓，是比较秀气的，与之前认识中的印象相当贴近，配上江边潇洒延绵的山岩，很多地方都是很具有特色的那种漓江风景。下午天色由睛转阴，在九马画山那里坐了竹筏顺流而下，一边观赏沿江风景，快到兴坪码头，已经下起雨来了，雨中的漓江，远山隐现在雨雾中，近处雨打江面清风骤来，迷迷朦朦，与晴天气下的明媚旖旎相比又是一番不同的风景，于是和朋友多拍了一些雨中的漓江风景照片。

总之，去阳朔，漓江是不能不游的。

旅行的意义

3天的阳朔游行程很快就结束了，第3天下午坐车回来，回到熟悉的广州平淡地工作生活。

有人说，所谓旅行，就是从一个自己活腻的地方到一个别人活腻的地方，说的也有一定的道理，与《在路上》那种任风逐浪随波而流的长期旅行不同，当下我们的旅行方式更多的是那种匆匆而过点到即止，犹如在办公室的午间小睡，很忙的早班过去，一尝小憩，只是为了更清醒地继续迎接下午的忙碌。在平时繁忙的工作抽出一点时间，体验一下在某个陌生的地方出游，感受一下那种自由无羁的感觉，也算是一种惬意的事，出行之后再回来原来的地方，回到熟悉的生活，看待一些事情会有一种重新认识的感觉，熟悉的生活仍在继续，仿佛又“腻”回原地，但无论旅行与否，一个人的生命的经历只有一次，我们又何曾“腻”过？

MySQL Test Suite

Sep 1st, 2011

1. 介绍

在寻找MySQL测试程序的过程中，发现它的源代码中就自带测试程序，一套叫做MySQL Test Suite的包括二进制文件，脚本和测试用例，输出结果的测试框架。
关于这个MySQL Test Suite，《深入理解MySQL》有详细的描述，参考
http://book.51cto.com/art/201001/180745.htm
和
MySQL自动测试框架介绍，http://rdc.taobao.com/blog/cs/?p=480

2. 使用

2.1 系统apt源安装包中的Test Suite

通过apt安装的mysql-server，这个mysql-test-run.pl会被安装到系统目录的
/usr/share/mysql/mysql-test/
运行此脚本会报错 

$ ./mysql-test-run.pl
Logging: ./mysql-test-run.pl
mysql-test-run: *** ERROR: Could not find any of /usr/share/client_release /usr/share/client_debug /usr/share/client/release/ /usr/share/client/relwithdebinfo/ /usr/share/client/debug/ /usr/share/client /usr/share/bin

此脚本会假设../client目录下有mysql客户端程序，而通过apt安装的mysqlclient的目录layout与它的源代码目录的layout不一致，因此不能正常工作。

2.2 源代码库中的Test Suite

2.2.1 下载源代码

于是直接下载MySQL的源代码。在MySQL的官方网站上
http://dev.mysql.com/downloads/
点击社区版的”MySQL Community Server”下载链接，这时最新的版本是5.5.15，在平台选择下拉列表中选择”Source Code”，再选择Generic Linux TAR Archive包开始下载。

2.2.2 编译源代码

安装步骤和过程参考MySQL 5.5的官网文档
MySQL 5.5 Reference Manual, 2.9.2. Installing MySQL from a Standard Source Distribution
http://dev.mysql.com/doc/refman/5.5/en/installing-source-distribution.html

在linux平台下编译要注意，与5.5之前的版本用GNU AUTOCONF和AUTOMAKE编译不同，版本5.5将各个平台下的编译工具统一为CMake(CMake，Cross-platform Make是一个强大的跨平台开源编译工具)。所以编译的步骤有一点不同。

对比MySQL 5.1 Reference Manual，5.1的编译步骤是：
(参考http://dev.mysql.com/doc/refman/5.1/en/installing-source-distribution.html)

# Beginning of source-build specific instructions
shell> tar zxvf mysql-VERSION.tar.gz
shell> cd mysql-VERSION
shell> ./configure --prefix=/usr/local/mysql
shell> make
shell> make install

而版本5.5编译步骤是：

# Beginning of source-build specific instructions
shell> tar zxvf mysql-VERSION.tar.gz
shell> cd mysql-VERSION
shell> cmake .
shell> make
shell> make install

当然要顺利编译成功，需要安装cmake，在ubuntu下通过apt源安装cmake很方便:

bash# aptitude install cmake

在ubuntu下编译cmake那一步遇到问题，报错如下：

-- Check size of wint_t
-- Check size of wint_t - done
-- Could NOT find Curses (missing: CURSES_LIBRARY CURSES_INCLUDE_PATH)
CMake Error at cmake/readline.cmake:83 (MESSAGE):
Curses library not found. Please install appropriate package,

remove CMakeCache.txt and rerun cmake.On Debian/Ubuntu, package name is libncurses5-dev, on Redhat and derivates it is ncurses-devel.
Call Stack (most recent call first):
cmake/readline.cmake:127 (FIND_CURSES)
cmake/readline.cmake:217 (MYSQL_USE_BUNDLED_LIBEDIT)
CMakeLists.txt:257 (MYSQL_CHECK_READLINE)

-- Configuring incomplete, errors occurred!

发现它依赖于ncurses库，于是通过apt源命令行安装:

bash# aptitude install ncurses-dev

ncurses-dev是一个虚包，实际安装的包是libncurses5-dev。然后清理之前出错的临时输出:

bash$ rm -f CMakeCache.txt

这一步相当于”make clean”。然后重新编译

bash$ cmake .
bash$ make
...

2.3 简单使用介绍

编译完源代码之后，可以直接运行mysql-test-run.pl，切换到源代码根目录下的mysql-test目录:

bash$ cd mysql-VERSION/mysql-test

通过命令行

bash$ ./mysql-test-run.pl --force

自动启动mysqld并运行所有测试用例，–force选项相当于make的–keep-going选项，不加此选项的默认情况，mysql-test-run.pl脚本在首次遇到测试错误时就会中止，不运行后面的测试用命，添加此选项后即遇到单个错误也不会立即中止，继续运行后面的测试。
如果需要有选择性地只执行部分测试用命，可通过–suite选项指明需要执行的测试类别，如输入命令

bash$ ./mysql-test-run.pl --suite=main

只执行目录t下的所有测试用例，还可以通过指明单个测试名来执行单个或多个用例，如输入命令

bash$ ./mysql-test-run.pl alias analyze

只执行alias.test和analyze.test两个测试用例。 对已经启动的mysql服务器(mysqld)进行测试，可使用命令：

./mysql-test-run.pl --suite=main --force --extern sock=/var/run/mysqld/mysqld.sock --extern user=test_user --extern password=test_user

其中–extern用来将特殊选项的值传入，sock指向mysqld启动打开的sock，user和password分别指定连接数据库服务器的用户名和密码。
