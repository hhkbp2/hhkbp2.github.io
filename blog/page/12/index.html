便，我回来了

Mar 21st, 2010 | Comments

很快4月就要迎来ubuntu 10.04，时间过得飞快，不知不觉就是2年，8.04的发布似乎还如在昨天——当然是一个夸张的说法，感觉是这两年来生活与经历变了许多（人也折旧了好多），但PC上的8.04却一直稳如磐石，每天都跑得欢欢畅畅，没出过毛病，也可能是因为我只升级LTS的包吧 。很多软件包都旧了，想给我的PC升升级了。

ubuntu的版本支持时间很有限，即使是LTS，桌面版本也只支持3年，3年之后退役之时怎么办呢，唯有升下一个版本了。而版本升级却不是那么平滑，难免或多或少有问题。虽然我没做过版本升级，不过论坛上有做小白鼠牺牲的也不是一个两个了。不过能用3年的人估计也少，都是有新版本来了就立马升级（重装）上去，之前在论坛就有帖子问有没有人一个版本连续用2年，跟帖的人不少，却没有一个真的用到2年。

今天再回过头来看看，平滑升级真的是一个很好的特征。与其几年一次重装，不如每天一坨，稳定地升级上去。稳定，对于“漂”在外面的我很有杀伤力。

于是重回Debian，重装系统。PC上的硬盘太老，经过多年摧残，坏道多多。而上面的叉p常常是几个月都不用一次，干脆全部格掉。用gnome testing的镜像安装好后，简单升级配置一下，就已经很好用了。界面虽然简洁了一点，不过胜在速度快，跑起来快很多，即使iceweasel也比旧的firefox快，简单比较了一下，安装的包少了500多个，没有深究是哪些包。看来还是大便对于渐渐变成命令控的我更有吸引力些。才发现还有debian-reference-zh-cn这么好的文档。

Color Theme Darkmate for Emacs

Mar 1st, 2010 | Comments

Color Theme is an Emacs-Lisp package for making Emacs colorful and beautiful. It is popularly used among Emacsers, including me. When I use it and try to fontify Emacs as the font-lock facilities do, I found there is a bug in the Color Theme latest version 6.6.0 package.

Line 689 of the source file “color-theme.el”, in the defination of function “color-theme-spec-compat”, it uses “cadar” to fetch out the property of only the first element of face specification, rather than fetch out the whole specification. So give the face specification as

(dired-mark
      ((((class color) (min-colors 88)) (:foreground "red"))
       (((class color) (min-colors 16)) (:foreground "green"))
       (((class color) (min-colors 8)) (:foreground "blue"))
       (((type tty) (class mono)) (:foreground "white"))
       (t (:foreground "red"))))

It is a correct face specification(refer to macro `defface’ for more information of face specification), and “dired-mark” should be in color “green” under 16-color environment and “blue” under 8-color environment. However function “color-theme-spec-compat” just set “dired-mark” to be “red” in any case. It reduces the flexibility of face-lock facilities of Emacs when using Color Theme.

To fix this, I revise the defination of “color-theme-spec-compat” like

(defun color-theme-spec-compat (spec)
  "An redifination of `color-theme-spec-compat' of `color-theme'.
To apply different face settings according to different term types."
  (let (compat-spec props)
    (dolist (dis-atts spec)
      (setq props (cadr dis-atts))
      (when (plist-member props :bold)
        (setq props (color-theme-plist-delete props :bold))
        (unless (plist-member props :weight)
          (setq props (plist-put props :weight 'bold))))
      (when (plist-member props :italic)
        (setq props (color-theme-plist-delete props :italic))
        (unless (plist-member props :slant)
          (setq props (plist-put props :slant 'italic))))
      (setcdr dis-atts `(,props))
      (add-to-list 'compat-spec dis-atts 'APPEND))
    compat-spec))

and put this into the file “color-theme+.el”.

I give a try out to customize a color-theme of my favour as well. I name it “color-theme-darkmate”. This color theme is inspired by the theme darkmate of gEdit, which is a color theme somewhat similar with the theme of editor TextMate on Mac. I never use the editor TextMate before and don’t know how much does “color-theme-darkmate” look like the theme TextMate. The settings in “color-theme-darkmate.el” is created in the opinion of using a few elementary colors as consistantly as possible in various modes to make a colorful and eye candy Emacs.

You could download “color-theme+.el” and “color-theme-darkmate.el” here.

Enjoy!

Change Eclipse Start-up Splash to Your Favour

Jan 27th, 2010 | Comments

很久之前就知道如何替换eclipse启动时显示的splash(闪屏)了，不过一直没有文档记录。这里整理一下。

1. 更换默认的splash

1) 在命令行参数加上

-Dosgi.splashLocation=/opt/eclipse/splash.bmp

或在eclipse根目录(对于网上下载的版本是根目录，对于linux各发行版下安装版本，请参照各发行版对应eclipse包)下的配置文件eclispe.ini中加入上面一行。意即指明启动splash的地址。定义osgi.splashLocation，会覆盖osgi.splashPath的定义，即用自定义的splash替换默认的那个。

2) 在eclipse/configuration/config.ini中加入一行

osgi.splashLocation = ${你的splash路径}

如在linux下有

osgi.splashLocation = /usr/lib/eclipse/YUKI.bmp

(上面的/usr/lib/eclipse/即为ubuntu 8.04下的含有eclipse.ini的目录)

在windows下有

osgi.splashLocation = D:/software/eclipse/splash.bmp

3) 直接替换原来的splash图片文件

参考在config.ini中osgi.splashPath定义，找到默认splash图片文件在文件系统中的位置。如在ubuntu 8.04下的config.ini有

osgi.splashPath = platform:/base/plugins/org.eclipse.platform

于是可以在

/usr/lib/eclipse/plugins/org.eclipse.platform_3.2.2.r322_v20070117b

找到默认的splash文件，splash.bmp。直接用你喜欢的图片替换掉此文件即可。

效果见截图：

2. 制作个性的splash

由于只是简单地替换掉背景文件，而进度条和loading信息没有改变，所以最好还是使得自己制作的splash与默认的大小455×295一致，否则进度条与图片边缘不一致会比较难看，个人建议至少要使得宽度一致，而图片的格式应为.bmp文件。在网上遇到喜欢的图片资源，可以使用GIMP来编辑一番最后再转为.bmp文件。至于图片内容，就随便随便啦－－例如原纱央莉的……Amitabha(向如来致敬)，罪过罪过，就此打住，不要给小朋友造成坏影响。

给emacs添加多种配色主题

Jan 12th, 2010 | Comments

在读写代码时workspace有个好的配色，能让编程过程变得生动多彩，也能提高效率。emacs 23的默认配色主题也不错，不过个性张扬的hacker们当然不满足于只有一个选择，很多emacs玩家喜欢自已定制主题，不过一点点地配置主题多少有点繁琐，在手工hack之前，可以先试下color-theme扩展－－它包含了50多个配色主题，让你的emacs更丰富多彩。在pluskid wiki上的color-theme介绍中可以看到一个截图，是不是很漂亮呢。当然也可以参考color-theme on Emacs Wiki, 到其项目主页下载最新版本，现时为6.6.0, 参考包中README文件安装。将color-theme.el和theme文件夹放在loadpath下，然后在.emacs中加入

(require 'color-theme)
(color-theme-initialize)
(color-theme-midnight) ;;选择你喜欢的主题

其中在最后一行选择你喜欢的主题，我暂时在用的是midnight。也可以通过

M-x load-library RET color-theme

(其中RET为回车键) 加载color-theme, 然后通过

M-x color-theme-select RET

进行选择。在光标所在主题上按i或l键为应用此主题，按d键输出调用主题的命令(即为写入 .emacs 最后一行那个)。

对于喜欢 TextMate主题的朋友，这里另外分享一个类似 TextMate风格的配色，见：

http://blog.jdhuntington.com/2008/11/emacs-color-theme-blackboard.html

我最早接触类似的风格是gEdit的DarkMate主题(长什么样？截图看 这里)，然后一发不可收拾，现在用的source insight, Ultra Edit, Eclipse, …(虽然前两者为盗版)全部自定义变成类似配色，可见其魅力之大。厉害吧，呵呵(换一种碟形世界的说法－－变态吧，呵呵)。

PS: 关于 gEdit 的配置文章可参考我很久之前在 ubuntu中文论坛 的一个帖子: gEdit: 打造简洁，小巧的编程环境，gEdit 用做一般文本编辑其实也不错，简洁快上手。

如何全屏emacs

Jan 11th, 2010 | Comments

很多像我一样喜欢将 emacs 作为全 workspace 的人都想喜欢将 emacs 全屏显示(只将 emacs 留下，其它桌面内容全部去掉，为此我错过了n多朋友在 QQ 或 pidgin 上的呼叫，sorry~~)。这里分享一下emacs全屏的配置：

linux 平台

在 linux 下，可以通过向 X server 发送全屏消息，在 .emacs 中写入

;全屏
(defun my-fullscreen ()
  (interactive)
  (x-send-client-message
   nil 0 nil "_NET_WM_STATE" 32
   '(2 "_NET_WM_STATE_FULLSCREEN" 0)))

;;最大化
(defun my-maximized-horz ()
  (interactive)
  (x-send-client-message
   nil 0 nil "_NET_WM_STATE" 32
   '(1 "_NET_WM_STATE_MAXIMIZED_HORZ" 0)))
(defun my-maximized-vert ()
  (interactive)
  (x-send-client-message
   nil 0 nil "_NET_WM_STATE" 32
   '(1 "_NET_WM_STATE_MAXIMIZED_VERT" 0)))
(defun my-maximized ()
  (interactive)
  (x-send-client-message
   nil 0 nil "_NET_WM_STATE" 32
   '(1 "_NET_WM_STATE_MAXIMIZED_HORZ" 0))
  (interactive)
  (x-send-client-message
   nil 0 nil "_NET_WM_STATE" 32
   '(1 "_NET_WM_STATE_MAXIMIZED_VERT" 0)))

即可通过执行命令

M-x my-fullscreen

全屏。其中也包含最大化窗口命令 my-manimized() 定义。不过有个问题，全屏后切换到scim五笔输入法后会闪屏不断。

这部分来自ubuntu中文论坛的帖子Emacs 最大化和全屏的配置，感谢kemean的分享。

windows平台

之所以有这部分是因为我也经常在公司的xp使用EmacsW32。可以通过安装darkroom扩展来去掉标题栏，以达到全屏的效果，参考

http://www.martyn.se/code/emacs/darkroom-mode/

内有安装与使用说明。不过我实际使用感觉并不好，w32下最大化足已。
