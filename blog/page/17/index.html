本来无一物，键盘惹尘埃

Jun 19th, 2011

读《六祖坛经》

最近找了一本《坛经译注》，像蚕食桑叶慢慢的一句一句一页一页很认真的读，很早之前就知道六祖的名句”本来无一物，何处惹尘埃”，但是直到最近才比 较系统的详细的了解六祖的思想。手上的这本《坛经译注》是中华书局在2010年出的。《坛经》是六祖惠能的言行录，据说是六祖的弟子法海记 录整理出来的，辗转流行的过程出现了很多个版本，此书选用的是文字比较浅显的元代宗宝本，一巻共10品，记录了惠能的出身，经历，主要的思想体系，传法的 过程等，既有抽象的概念，也有一些故事。我是乐于读故事的。在说理中穿插故事肯定不是由佛经首创，庄子就很长于此道。故事当然是为说明佛经中的道理而设 的，记得之前看过一本专门收集佛经故事的书，没有那些佛理作为中心，只读故事，估计只能了解到比较有娱乐性的皮毛部分吧，感觉类似去看现代派画展只看裸 女。

这本书目前还没读完，不过里面说的主要的思想算是有点明白了，简单的说就是:

世界的事物和现象是不断 变化的，是虚幻而不确定的存在，而在所有现象和变化的背后是世界内在的规律性和永恒不变的本质，要透过多变的现象把握到世界本质的真理性。人生不过是一个 短暂而多变的过程，不要被为声色名利这些外在的事物或欲念困缚，对外不刻意追求，对内不执着，不故意为善也不能放纵行恶，不会因为厌恶的东西而烦恼，对于想 要追求的东西也保持一种平常心，最重要的是认识自我，明了自己的本性，保持一种顺其自然随缘任运的心理状态和生活状态。

这只是我对《坛经》中部分内容的理解，没有覆盖到书中全部内容。

宇宙广阔浩大，太阳照常升起。保持一种禅的心境，我觉得，对于现代人看待自己的生活，也是有帮助的。至少能让我们在追求的过程少一份狂热，多一份平常心，在日常生活中多一份对世界对自然的敬畏，至少让我们不那么容易变成痴狂，变成过劳病或过劳死。

而 回头想想这几年，内心可谓是不平静，常常活在一种放逐一般的焦躁之中，也许是因为在读《坛经》，最近发现内心渐渐趋向平和宁静，有些东西看轻了不再成为烦 恼，有些东西继续追求但是渐渐有了一种平常心，说到这里想起不久看过的胡正的寻人启示(他是linux平台翻译软件stardict的作者)，他应该是醉 心于佛学的。佛祖说四大皆空，不知成佛一事为不为空？愚以为，不狂热于追求成佛，才能走上佛道啊。

一份缺少诚意的中式什果沙拉

May 16th, 2011

——评《Python UNIX和Linux系统管理指南》

手上这本《Python UNIX和Linux系统管理指南》(以下简称py4sa_cn，py4sa是对应的英文版《Python for UNIX and Linux System Administration》的缩写，cn表示中文译版)从卓越亚马逊网购拿到已经有2周，不过一直只是搁在书桌上没有看，这几天有空的时候随手翻看了一下，到今天14日这一本薄薄的400多页的书已经全部看过一遍，看完掩卷，感觉就像在饥肠辘辘的情况下刚刚吃过一份缺少诚意的中式什果沙拉。

《py4sa_cn》这本书主要介绍Python作为一门简单易用，具有强大库功能支持的语言如何应用到管理UNIX和Linux服务器上，正如书的封底《Red Hat Magazine》的评价所说，是针对Python的入门者，读者不需要有Shell编程经验，其实这个定位其实比较尴尬，因为这样一本书的读者其实主要有两种，一种是有UNIX/Linux系统管理的工作需要或经验，一种是Python的爱好(学习)者，当然两种读者可能会有交集，而前者不会没有Shell编程背景，也不会满意于系统管理或编程方面入门级的唠叨，而后者(我也是其中一个)更多的在于扩充知识面，了解Python与系统管理这两者的结合点，这本书也不会是学习python的第一本书，所以没必要从零开始讲(如详细讲述str的方法)，而《py4sa_cn》给人的感觉就像是它要做第一种读者的第一本Python书，或者第二种读者的第一本系统管理书，而这两个目标都完成得不好，在这点上个人觉得不够给力。

书中内容的确讲得很浅白，涉及的范围面很广，不过每个方面都是蜻蜓点水，简单介绍一下背景和使用，加插几个例子，缺少深入的内容，只可以作为起点扩大视野，从书中给出的线索开始找其它资料代码进一步扩展和深入，从英文版原书两位作者的背景介绍来看，两位应该都是牛人，书中的内容涉猎范围的确很广，体现了两位作者的功力，对于我这样一个菜鸟来说，就像是刘姥姥走了大观园，各种的模块框架术语让我目不暇接眼花瞭乱，但是定位之低内容之浅以及为数众多的行文前后索引和排版错误，又让人觉得它像是匆忙赶工的书稿而非成品。O’Reilly官网这本书的页面上有一个勘误表，不过很明显并非大部分的错误都收录了进去。到了中文译版，这些错误并没有被发现和修正，反而有更多的新问题被引入。我读书没有写读书笔记的习惯，不过对于写得好自己喜欢的书，如果看到有错误一般都会记录下来，做一个勘误表，像之前的Cpp Primer勘误或UNPe3勘误，不过这本书的话，看来觉得意趣廖廖，搞得没有列勘误表的兴致了。

举几个例子说明在阅读《py4sa_cn》过程中发现的问题：

1. 行文印刷或排版错误类

1.1 第43页最后一句

“%wquickref中的%ho帮助行同使用标准的%who?返回”
这里可能是张无忌大教主来做过一次乾坤小挪移，正确的应该是
“%quickref中的%who帮助行同使用标准的%who?返回”

1.2 第102页第2段第1句

“Finditer()是对findall()的略微修改”
这里有microsoft office word的影子，Python的标识符可是大小写敏感的。

1.3 第116页，”StringIO”一节下面第2段

”’并将字符串”This is…\nnreadline()should see””’
有错，应该是”\nreadline() should”。

1.4 第124面，”Element Tree”一节下，示例代码Out[3]中，后面的应该是没有的
1.5 第249页名人简介框框，第1句

“Kevin Gibbs是Google App Engine中的先进技术。Kevin在2004年加入Google”
这个…估计Kevin Gibbs本人看到会震精吧。

1.6 第392页，”Unix Mashups: 整合Shell命令到Python命令行工具中”一节第2段第1行

“另外，混合一个命令行工具和纯Pyton”
明显拼写错误了。即使完全不会技术的审校也轻易能看出来，除非他不知道在审的书是什么书名。

2. 代码错误类

2.1 第155页，第3个代码示例中，命令行
'''
$ python port_checker_tcp.py -a 192.168.1.15 -p 81 && echo "FAILURE"
'''

其中的”&&”应该是”||”才对，英文版原书也是”||”，估计从原书copy然后paste的时候搞混了，向广大中小学生证明了一点：其实抄或者说复制粘贴也是需要专业的精神的，否则一不小心抄错了或者把别的同学的名字也抄下来了，敢情闹笑话不是。

2.2 第383页，以例13-1 sysargv.py的代码实现，来敲命令行
"./sysargv.py foo"

命令行是不可能出现

"['./sysargv.py', 'test']"

的，看来张教主又出现了。这个问题英文版原书也有。

3. 硬译乱译类

3.1 第53页，”可变扩展”一节中第1段第2行

“其中，第一个是可变扩展（Variable Expansion）”
第1次看到这个句子的时候我愕然了一下，然后看看标题是大黑体的”可变扩展”，不禁哑然失笑。我不想猜测译者是怎么翻译这本书的用的是哪种工具，不过如果真的要猜测的话，那么出现在我脑海中的第一件事物就是自动翻译软件，如google翻译或金山快译。Variable Expansion这里面的Variable不译成变量(名词)而译为可变的(形容词)，我觉得真是相当突兀的，用时下流行的话来说，就是被雷到了，而且是里嫩外焦那种。

3.2 第109页第3段第2行
'''使用"try/finally"代码块将文件进行封闭是一个非常有用的方法，尤其是在使用write()调用时。'''

这里我前后来回看了几次都不明白什么叫做”将文件进行封闭”，直接找来原文，原来是

'''It is good practice to wrap your file opens with a try/finally block, especially when you are using write() calls.'''

姑且不论这个”将文件进行封闭”是什么意思，那个”非常有用的方法”是怎么来的呢?难道”好的实践”这么直白的意思还要意译？

3.3 第186页，”使用路径，目录和文件”一节第2段第1行

“通常，Python的标准库有一些杀死工具，可以停止作业的执行。”
“杀死工具”一词看得我半荦半素，直接找到原文
“As always, the Standard Library in Python has some killer tools to get the job done.”
What？原来如此！难怪前后文都没有说到kill进程，但突然冒出这样的一句。又见自动翻译软件！其实它的意思是Python的标准库中有一些杀手级工具，可以帮你完成工作，这里的工作结合上下文来看是指进行与路径，目录和文件相关的操作。

3.4 第191页倒数第2段第3行

“注意，line[7]之所以返回了file11.txt，是因为dirB是dirA的超集。而在line[6]中之所以结果为空，是因为dirA包含了dirB的全部内容。” 这样一来相当于睁大眼睛说瞎话了，英文版原文是
“Notice that line [7] returns file11.txt because dirB is a superset of dirA, but in line [6] the results are empty because dirA contains all of the same items as dirB.”
其实很简单，dirA和dirB是两个集合，dirA是dirB的子集，dirB是dirA的超集，两者分别为

In [3]: dirA
Out[3]: set(['file1.txt', 'file00.txt', 'file0.txt'])
In [5]: dirB
Out[5]: set(['file1.txt', 'file00.txt', 'file11.txt', 'file0.txt'])
3.5 第302页第一行

“在例10-16中，我们告诉tracd在screen中运行。一旦进程开始运行，可以简单地使用Ctrl-A进行分离。如果希望重新连接，使用Ctrl-D。” 这句话太强大了，一下就把我这个GNU screen小资深用户搞迷糊了。来看英文版原文
“In Example 10-16, we tell tracd to run within screen. Once the process starts, we can then simply detach using Ctrl-A, then Ctrl-D, if we ever want to reattach.”
意思是说，用screen跑进程可以先detach，什么时候想回到这个环境再reattch，detach的话用的是Ctrl-A后接Ctrl-D这两个快捷键组合，其中Ctrl-A是screen默认配置下的控制键组合的前缀。翻译出来的中文真是断句如神强大无匹七零八落。

4. 前言不搭后语类

4.1 第397页，”整合配置文件”一节下面第3段最后一句

“请参阅前一章以获得更多的使用ConfigParser模块的细节信息。”
而前一章根本就没有讲到ConfigParser，这个错误来自英文版原书。

其实还有很多问题，比如代码示例没有标注源文件文件名，不方便找到对应的源码文件；示例代码缩进乱排乱摆犹如街上的走鬼小摊，要知道Python是一种以缩进来表示代码结构的语言；提供下载的代码也显得相当敷衍，比如

code/code/data/code/enumarate_file_dir_path.py
code/code/data/code/enumarate_paths.py

这两个文件其实代码是一样的；排版英文字符间距大小不一；代码示例中命令行有时有命令行提示符”$”，有时没有提示符，显得相当随意；纸质太差。

上面举例列出的问题并不是我在阅读《py4sa_cn》过程中发现的全部，在更多的读者的雪亮眼睛面前，相信也将会有更多的错误被发现。这种种问题都显得这本书的出版质量与O’Reilly的大名相去甚远，有的是印刷排版的小错误，但是有的地方体现出来的，更多的是缺乏好好做一本书的诚意。总的来说，这本书的将Python与系统管理结合角度新颖，覆盖范围也比较广泛，虽然内容过浅，也算是一本值得一读的入门级轻松读物，一碟开胃小菜，就像一份什果沙拉，里面的鲜果种类不少，但吃起来味道比较淡，加上来一个毁一个来两个毁一双的中式做法，最多也只能算是一份缺少诚意的中式什果沙拉。建议直接读英文版，中英文两个版本的电子书在网上都很容易找得到，至于中文版的纸版书是否值得一买，这就不好乱下评价鼓惑读者了，各人有各自的判断，作为一个支付真金白银买了这本纸版书的读者，我在这里分享一下自己的感受，并衷心表达两点希望：1. 世界和平；2. 码出来的字对本文读者有所帮助。吐槽完毕。
