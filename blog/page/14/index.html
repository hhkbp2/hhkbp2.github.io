Darkmate主题

Mar 21st, 2012

darkmate是一个gedit主题的名字，我用过之后觉得不错，后来将这个主题的配色移植到了Emacs。

Emacs下的主题多用Color Theme扩展来管理，它的介绍文章请看这里。我在使用这个扩展来定制配色的时候，发现在它的6.6.0版本下有一个问题：只能准确地指定GUI下的配色，终端下配色与GUI相差较远。看过源代码后，确认是代码有一个bug，于是做bug fix写了color-theme+.el(详细参考这篇文章)，同时也给Color Theme的原作者报了bug。

有了color-theme+.el，就可以既准确地定制GUI和终端下的配色，于是我将darkmate也做成了一个Color Theme主题文件。由于配色较多，我把各种配色按不同的内容分开成多个不同的文件。

所有配色相关的代码文件列表(按字母序排序)及各自对应的内容如下：

calendar-face-settings.el   ---> calendar
basic-face-settings.el      ---> 基本配色
color-theme-darkmate.el     ---> Color Theme主文件
color-theme+.el             ---> 前面所述的bug fix
color-theme-settings.el     ---> Color Theme相关配置
dired+-face-settings.el     ---> dired+扩展
ecb-face-settings.el        ---> ecb
emms-face-settings.el       ---> emms
erc-face-settings.el        ---> erc
flyspell-face-settings.el   ---> flyspell
ido-face-settings.el        ---> ido
isearch-face-settings.el    ---> isearch
jde-face-settings.el        ---> Java开发
linum-face-settings.el      ---> linum扩展
paren-face-settings.el      ---> 括号配对
tabbar-face-settings.el     ---> tabbar

要在emacs中加载darkmate配色，到这里下载上面的文件后，需要在dot emacs文件中添加

(require 'basic-faces-settings)
;; 加载Color Theme扩展
(require 'color-theme-settings)
;; 加载定制的darkmate主题
(require 'color-theme-darkmate)
(color-theme-darkmate)

即可加载主要的配色配置。对应其它扩展的各个文件，可以根据各人的使用需要另外添加，如使用erc的人可以在dot emacs文件中添加

;; 加载erc配色配置
(require 'erc-face-settings)

而使用JDE来做Java开发的人可以添加

;; 加载jde配色配置
(require 'jde-face-settings)

Gnome-shell扩展管理

Mar 4th, 2012

之前写了两篇小文：

《杯具的诞生》
《折腾gnome-shell》

介绍了安装gnome-shell之后的一些使用情况。这里继续向大家介绍一下，不久前上线的gnome-shell扩展的官方网页：
https://extensions.gnome.org/

在这个页面里，按功能划分了几个tab：扩展的浏览/索引查找(打开网页的默认tab “Extensions”)，已安装的扩展的启用/禁用/卸载等管理(tab “Install extensions”)，注册帐号登录(tab “Add yours”)。页面风格一如gnome继往的简洁明了，也许是因为上线没多久，目前提供的功能不多，所以网页标题里打上了”Alpha”字样，但使用起来的感受比较好，相对于之前gnome-shell扩展没有一个统一管理的界面，这个网站的出现大大地提高了用户体验。从使用方式上来看，比较像debian/ubuntu的软件管理器(中心)，把所有可用的包统一到同一个界面，提供了安装/卸载，或启用/禁用的功能，于是查找/试用gnome-shell的扩展变得非常简易。

这里收录了数量众多的扩展，比较赞的一点就是，从gnome 3.2之后，安装了gnome-shell之后，只要安装了”gnome shell integration”这个包，就可以直接在网页上来安装扩展了，类似于用firefox在firefox的扩展官方网页上安装扩展，看到喜欢的扩展，点击一下，扩展包就自动下载到本地，并自动安装好，至于版本兼容，安装到哪个目录这么些问题，不用用户折腾处理，实在很省心吧。比firefox的扩展管理网页更好的是，gnome-shell的扩展网页还提供了按钮来启用/禁用/卸载按钮，扩展管理起来相当方便。相对使用本地客户端”gnome-shell-integration”来启用/禁用扩展的方式，这个网页版的功能更强大，也更方便。

接下来show一下目前我安装的使用gnome-shell扩展吧，按字母序排：

  1. advanced settings in user menu
    在应用程序菜单，添加有”木偶”图标的”Advanced Settings”项，打开就是gnome-shell-integration的配置界面，见下面图中的红字1标出部分

  2. Application Menu
    将gnome-shell默认的任务栏左上角”Application”按钮，换成老版gnome应用程序菜单按钮，点击打开应用程序列表，见图中红字2部分

  3. Auto Move Windows
    应用程序创建新窗口时，移动到特定某个workspace，没有图形的配置工具，用得不多

  4. Battery Percentage Indicator
    电池电量指示器，对笔记本特别有用

  5. Dock
    给桌面添加一Dock，Dock里面的内容同Overview模式下左边的Dash一致，感觉与Dash有点重复了

  6. gTile
    功能类似awesome，平铺式窗口布局管理器，用得比较少，暂时我还没摸熟用法，作为一个插件，比起awesome来要轻量不少

  7. Lock Keys
    在面板上显示Numlock和Capslock的状态，在没有键盘灯的情况比较有用，对像我一样交换Capslock和其它键的也好用

  8. Media player indicator
    在面板上添加音乐播放器的指示图标，这个在老gnome上也有了，支持Rhythmbox和amarok，但不支持audacious，有点遗憾，因为我常用后者

  9. Move Clock
    将时钟从顶部面板中间位置移动到右边，风格像老的gnome，时钟format可以通过dconf配置，这个扩展是必不可少的，从截图中可以看到效果

  10. Music Integration
    在通知栏上添加音乐播放器的整合，类似于前面的8

  11. Native Window Placement
    令Overview模式下，窗口的预览更紧凑一些，而不是每个窗口都同一大小

  12. Panel Favorites
    将Overview模式中Dash上的快捷方式放到顶部面板，提供了类似老版gnome的面板快捷方式，必不可少的一个扩展，见上面图中的红字12部分

  13. Places Status Indicator
    在面板上添加目录快捷访问图标，就像老版gnome面板上的”Places”

  14. Recent Items
    在面板上添加最近访问的文件列表图标，就像老版gnome面板上”Places”菜单下拉出来的”Recent Files”

  15. Remove Accesibility
    删除面板上的”辅助”图标，这个扩展也是必需的，因为那个图标对很多人来说几乎没用

  16. Remove Panel App Menu
    去掉面板上当前打开窗口的指示图标，个人感觉如果没有底部面板，顶部面板有这个指示图标还是挺有用的，如果有了底部面板，像我就喜欢用tint2，就可以把这个顶部面板的去掉更清爽，效果见上图上红字16部分

  17. Shut Down Menu
    将顶部面板中用户名图标下拉的”Suspend”菜单项，改为”Shut Down”，在之前的文章我批评过gnome-shell默认安装下没有关机菜单的这一”反人类”设置，这个扩展是必需的

  18. Static Workspaces
    与tint2等底部面板搭配起来比较好用，像我一样不习惯gnome-shell默认的动态创建workspace的行为，可以试用一下

  19. Timer
    在顶部面板添加一个简单的面板小时钟，功能比较弱

  20. Trash
    在顶部面板添加回收站图标

在安装了一堆的扩展之后，搭配tint2，我机器上的这个gnome-shell的使用体验和老版的gnome就差不多相同了，只是有些小变化。在刚装上gnome-shell，没有装扩展的时候，我曾经觉得它十分的不好用，于是写了上面的第一个文章《杯具的诞生》，吐槽了一番，到今天gnome-shell的用户体验已经比当初好出很多倍。

对比来看，老版gnome更像是A餐B餐，软件包就那么几个，全部plugin用到的没用到的都绑在里面，装上之后，每个用户爱用哪个plugin就配哪个plugin，而gnome-shell更像是自助餐，用户喜欢用哪个再自行下载安装启用，而越来越多的扩展出现，提供了不断丰富的功能和操作方式，像从网页点击按钮直接安装扩展这种便捷方式就是老版gnome所没有的，从这个角度来看，gnome-shell灵活和定制性强的优势已经开始慢慢展示出来。

金杯月明夜，与君醉梦乡

Feb 19th, 2012

——读乔治.R.R.马丁的《肉院情人》

克劳修斯告诉我们，说我们生活所在的这个宇宙，是会热寂的，物体从高温会变低温，从有序变无序，总之一切都在衰老变坏，一去不返无可挽留。但这个宇宙中有很多事情看起来恰恰向着相反的方向发展，比如进化，又比如爱情。

小时候看粤剧《帝女花》，不明白为什么一对夫妻好端端的要一起喝毒酒寻死，问周围的大人也没问出个详细究竟——很多人听粤剧都并不那么深究内里的剧情。后来年岁渐长，慢慢积累多了书本知识人情世故，才懂得在那国愁家恨世事变迁之下，一份爱情最后也竟会变成两杯毒酒，一段唱词。剧中最惊艳的是，那红装妖娆的长平公主一手抛袖，一手甩袖，长衣飘舞，台上幽幽唱道，”落花满天蔽月光，借一杯附荐凤台上…“，教人不禁黯然动容，原来殉情也可以如此优雅动人。

爱情总是美好的，但美好的爱情亦难长久，特别是在乱世动荡之中。像我们一般的普通人，既不是生逢乱世，也没能经历跌荡起伏的人生，只是一个平凡得不能再平凡，在21世纪朝九晚五或者更糟糕的朝九晚九的小白领，又如何能遇上能如此美好，轰轰烈烈的爱情？倘若遇上太现实的女生，跟你说”有房有车吗？只要你是高帅富，我就喜欢”，也只能哑然失笑。纯粹的美好爱情，也许只能在大戏里面遥远地感受一下而已了。

所以说，无论什么时代，爱情都是很难寻得的。

看完《肉院情人》，我第一个想到的是，一个人一生中经历无数的恋爱分开，和一个人一生只经历一段或几段恋爱分开，然后进入死循环，轮回无数次，这两者有什么不同？

也许，乔治马丁老爷子给了我这个问题的答案。

试想一下，如果你活在某个未来的时代一个叫斯卡拉奇星的地方，要寻找爱情，你必须先要是个年轻小伙不懂爱也不懂情，不过却有养活自己的能力并且能够自由走动，但是要当故事主角的话，还需要本领高强经历曲折，苦个心智劳个筋骨什么的。而在这部《肉院情人》里面，主角的名字叫崔格。

每个年轻小伙都有第一次，当然主角也不例外，崔格的第一次，是在肉院妓馆里发生的，然后我们的主角，身为优秀控尸人，在漫长而空虚的时光中，内心却期待着爱情，渐渐成为肉院熟客，最后发现原来肉院并不是他的追求。

每个年轻小伙都有初恋情人，当然主角也不例外，崔格的初恋情人，是乔西，那个燃起他生命和激情火焰的女孩子，但乔西只希望与崔格做朋友。于是崔格依然孤独。

于是他离开，去了另一个星球，远方。在那里，崔格认识更多的人，走过了更多的地方，当然，也见到了更多的肉院妓馆。直到遇到下一段爱情。

下一段爱以“萝瑞”这个名字开始。萝瑞喜欢崔格，但也喜欢崔格的好友唐纳利，最后萝瑞选择了唐纳利，崔格出局了，然后他难过彷徨，前无未有的痛苦迷乱，于是他走得更远更广，最后他去了做之前他最鄙视职业，竞技斗场控尸人，有了夫人，成了一个不相信爱情的男人。

马丁老爷子的作品也看过多部了，在这部《肉院情人》里面，老爷子继续发挥着长抒情善说理的文笔，但非常少见的是，用了大段的笔墨来做性描写，来表达情爱的主题。文中对孤独，对爱情的思考也比较的独特与消极，笔下荒芜而苍凉。我想，老爷子年轻时必然也有过类似的孤独感，对爱情经过深深的迷惑与思考，不然那笔下如何能写得如此出色？很多写景的抒情的段落都写得很简洁清朗，但又寂寞萧索，如同主角的感情际遇，仿佛即使是最庞大喧嚣的城市，最明亮辉煌的建筑，也透着一股冷感。读完全文，我想这部《肉院情人》也许只告诉我们一个物理学认识，云雨交欢也许是对抗热寂的一种方法，但爱情不是。

那么爱情，到底是什么呢？

爱情，就像是满天月华夜的两杯毒酒；是控尸人世界的无数次相爱与背叛，亲密与分开；是苦追寻，求不得的无限轮回；是甜蜜的许诺，也是残忍的谎言。

因此，如果有一天你爱上了某个女孩，不妨对她说，爱情难得，欢笑亦少，漫长的夜晚之后，是白天和另一个夜晚，是广袤宇宙的热寂，我们为何不趁着良辰佳景，共醉今宵呢？

PS: 最后录一段文中的文字

“  ‘你很幸运。’有时他这么对她说…‘非常幸运。他们骗你说，世上有种东西叫爱情。他们教你去做傻乎乎的美梦，要你去相信梦，去追梦。他们告诉你，你，以及世上所有的人，都有命中注定的伴侣。但这是一派胡言。宇宙并不公平，从来就没有公平过…这是一个绝望的谎言，绝望的人们用它来互相欺骗，并且试图欺骗自己……‘  “
